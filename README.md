# 问道长生：HTML5 文字修仙

这是一个适合新手阅读的 Canvas 文字放置修仙游戏，支持 PC 浏览器和手机浏览器。

## 浏览器版本如何运行

最简单的方式是直接打开 `index.html`。正式部署时，建议把整个目录上传到任意静态网站空间，目录结构保持不变：

```text
index.html
platform.js
game.js
images/
```

本地开发也可以在项目目录启动任意静态文件服务，例如：

```bash
python -m http.server 8080
```

然后打开 `http://127.0.0.1:8080/`。

浏览器版本会自动适配：

- PC：游戏画布居中显示，支持鼠标点击、拖动和长按。
- 手机：画布铺满可用屏幕，支持触摸、滑动和长按。
- 存档：自动写入当前浏览器的 `localStorage`。
- 迁移：顶部“导出/导入”可备份或迁移 JSON 存档。

同一浏览器只有在“网站地址相同”时才会自动读取原存档。更换电脑、浏览器或网站地址前，请先导出存档。

## CloudBase 云存档配置

游戏采用“本地缓存 + CloudBase 云端同步”：所有修炼和战斗计算仍在浏览器完成。本地存档会先写入 `localStorage`，云端只保存完整快照；云端不可用时不会影响正常游玩。

部署前请编辑 [cloudConfig.js](cloudConfig.js)：

1. 填入 CloudBase 环境 ID；若控制台要求新版 Web SDK 发布密钥，同时填入 `accessKey`。
2. 在 CloudBase 控制台开启“匿名登录”，并把游戏部署域名加入 Web 安全域名。
3. 在数据库创建集合 `game_save`。本实现以 `player_xxxx` 作为文档 `_id`，确保一名玩家只有一条记录。
4. 按项目所用的匿名访问策略，为已登录的 Web 客户端开放此集合的读写权限。正式运营如需防止账号 ID 被猜测，建议把登录方式升级为微信/手机号等可跨设备验证的登录。

每条云端文档结构如下：

```json
{
  "_id": "player_8f92kd83",
  "userId": "player_8f92kd83",
  "saveData": { "完整 gameData 快照": true },
  "saveVersion": 42,
  "createTime": "2026-07-16T07:30:00.000Z",
  "updateTime": "2026-07-16T07:31:00.000Z",
  "updateTimeMs": 1784187060000
}
```

首次打开会生成并保存 `localStorage.userId`（形如 `player_xxxx`）。顶部“云存档”可查看账号、同步状态和最后保存时间，并提供立即保存、重新加载、删除云端副本和切换账号。换浏览器时，在新浏览器点击“切换账号”并输入旧设备显示的账号，即可优先读取该账号的云端进度。

同步规则：启动时比较本地与云端 `updateTime`，较新的完整快照覆盖较旧快照；首次无云存档时上传本地新档。普通挂机最多每 60 秒同步一次，后台、突破、轮回、获得装备或功法、完成宗门任务及调整轮回天赋会立即同步。`lastOnlineTime` 会跟随每次本地保存更新，进入游戏后现有离线结算逻辑以保存的时间戳补算自动修炼、洞府和镇守收益。

## 代码从哪里开始看

建议按下面的顺序阅读：

1. `game.js`：所有界面、按钮、修炼、升级、突破和存档逻辑。
2. `platform.js`：提供浏览器画布、鼠标/触摸、弹窗、生命周期和本地存储接口。
3. `index.html`：PC/手机浏览器入口和响应式布局。

## 当前数值规则

- 修炼一次：初始获得 1 点修为。
- 功法升级：初始消耗 20，之后每级费用翻倍；每级让点击修为 +1。
- 聚灵阵升级消耗灵石，费用公式为 `50 × (当前等级 + 1)²`；每级让自动修为 +1/秒。
- 外出历练每次有 60 秒冷却，可随机获得灵石或聚气丹。
- 聚气丹持续 60 秒，其间点击和自动修炼收益均为 1.5 倍。
- 境界包含凡人、炼气一至九层、筑基初中后期、金丹初中后期。
- 炼气九层突破筑基初期的成功率为 30%。
- 筑基后期突破金丹初期的成功率为 10%。
- 概率突破失败后，当前修为会全部清空。
- 所有进度通过 `platform.setStorageSync` 写入浏览器 `localStorage`。

## 图片资源

- 最终图片统一放在 `images/` 目录。
- 图片路径统一登记在 `game.js` 顶部的 `GAME_IMAGES`，玩法代码中不要直接写路径。
- 背景使用 `1280×720` JPG，人物使用 `512×768` 透明 PNG，妖兽使用 `512×512` 透明 PNG。
- 新资源加载失败时会自动回退为原来的纯文字界面，不会影响存档和数值逻辑。
- `unlockedImages` 只保存已经播放过的剧情编号，不保存图片对象。

## 开发者 GM 测试

`game.js` 顶部的开发开关：

```js
const DEBUG_MODE = true
```

开发时连续点击游戏标题“问道长生”5次，可以打开 GM 菜单。支持设置境界、修改资源、临时解锁系统、生成玄阶/天阶装备、生成随机功法和直接测试下一世。

快捷命令示例：

```text
/gm realm 元婴
/gm realm 化神
/gm stone 999999
/gm contribution 99999
/gm law 9999
/gm rebirth
/gm equipment
/gm technique
```

GM 会话开启后不会继续写入正式存档，也不会记录永久成就。刷新或重新打开游戏即可恢复进入 GM 前的正常存档。

正式发布前务必改为：

```js
const DEBUG_MODE = false
```
